# Workflow Expansion Round 3 Plan

이 문서는 3차 지식DB 확장 전에 후보 체인의 목적, 단계, 출처 기준, QA 검색어를 먼저 고정한다.
실제 데이터 추가는 후보별로 하나씩 진행한다.

## Selection Criteria

- 사용자가 바로 탐색, 검증, 정리 행동으로 옮길 수 있어야 한다.
- 고위험 영역은 진단, 법률 판단, 투자 판단을 직접 대체하지 않는다.
- 공식 기관, 학술 DB, 표준 문서, 공신력 있는 데이터 도구를 우선한다.
- 각 체인은 `정의 -> 근거 확인 -> 비교/분석 -> 실행 메모` 흐름을 유지한다.
- 각 후보는 5~8개 단계와 단계별 이론 링크, 검색어, 실행 도구를 갖춘 뒤 데이터에 반영한다.

## Candidate 1. 정신건강 정보 탐색 플로우

- 대분류: 의학·보건
- 중분류: 정신건강
- 목적: 증상이나 상황을 성급히 자가진단하지 않고, 신뢰 가능한 개념 확인과 상담 준비로 연결한다.
- 주의: 진단, 치료 결정, 위기 개입을 대신하지 않는다. 위기 상황은 지역 응급/상담 자원으로 즉시 연결한다.
- 대표 검색어: `정신건강 정보 탐색`

단계 후보:

1. 상황과 증상 기록
2. 핵심 개념과 용어 확인
3. 자가진단 도구의 한계 확인
4. 상담/진료 접근 경로 정리
5. 생활 환경과 위험 신호 구분
6. 상담 질문과 기록 템플릿 작성

출처 기준:

- 공공 보건기관과 전문 학회 자료를 우선한다.
- 일반 블로그, 광고성 상담 페이지, 단정적인 치료법 페이지는 보조 자료로만 다룬다.
- 위기 대응 자원은 지역성과 최신성이 중요하므로 실제 추가 시 최신 링크를 별도로 확인한다.

## Candidate 2. 규제 리스크 체크 플로우

- 대분류: 법·정책
- 중분류: 규제·컴플라이언스
- 목적: 서비스나 제품 아이디어가 부딪힐 수 있는 개인정보, 소비자, 광고, 플랫폼 규제 쟁점을 초기에 분리한다.
- 주의: 법률 자문을 대신하지 않는다. 실제 출시 전에는 담당 기관 자료와 전문가 검토로 이어져야 한다.
- 대표 검색어: `규제 리스크 체크`

단계 후보:

1. 서비스/제품 범위 정의
2. 데이터와 개인정보 처리 지점 확인
3. 소비자 고지와 약관 쟁점 점검
4. 광고/표시/마케팅 리스크 확인
5. 관련 기관 자료와 유사 사례 비교
6. 전문가 검토 질문 목록 작성

출처 기준:

- 법령, 정부기관 안내, 규제기관 보도자료와 가이드라인을 우선한다.
- 국가별 차이가 큰 영역은 관할 지역을 먼저 선택하도록 검색어를 구성한다.
- 법률 해석이 필요한 결론형 문구는 UI에서 피하고, 확인 질문 형태로 정리한다.

## Candidate 3. 바이오산업 기술 탐색 플로우

- 대분류: 농학·생명산업
- 중분류: 바이오산업
- 목적: 바이오 기술 아이디어를 논문, 특허, 적용 산업, 규제/윤리 쟁점, 사업화 단계로 나눠 탐색한다.
- 주의: 임상 효능, 투자 가치, 규제 승인 가능성을 단정하지 않는다.
- 대표 검색어: `바이오산업 기술 탐색`

단계 후보:

1. 기술 영역과 응용 분야 정의
2. 논문과 리뷰 자료 탐색
3. 특허와 기술 소유권 후보 확인
4. 적용 산업과 시장 사용처 정리
5. 규제, 안전, 윤리 쟁점 구분
6. 연구 단계와 사업화 단계 분리

출처 기준:

- 학술 DB, 특허 DB, 정부 R&D/산업 통계, 규제기관 자료를 우선한다.
- 기업 홍보자료는 시장 신호로만 사용하고 기술 검증 근거로 사용하지 않는다.
- 실제 데이터 반영 전에는 각 단계별 도구가 iframe 차단 대상인지 함께 확인한다.

## Implementation Order

1. `정신건강 정보 탐색 플로우`
2. `규제 리스크 체크 플로우`
3. `바이오산업 기술 탐색 플로우`

정신건강과 규제 리스크는 고위험성이 높으므로 데이터 추가 시 공식 출처 확인과 문구 톤 검수를 먼저 수행한다.
